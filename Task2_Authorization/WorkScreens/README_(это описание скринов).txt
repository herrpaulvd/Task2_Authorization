1)
В программе по сути есть три терминала:
	I FSMANAGER, управляющий загружаемыми в программу файловыми системами
	II FS, управляющий одной файловой системой. В его функции входит
		a. Авторизация пользователя
		b. Сохранения текущего состояния файловой системы
	III Собственно проводник файловой системы, который открывается после входа
	
Команда new в FSMANAGER создаёт новую файловую систему с одним пользователем, логин и пароль которого указываются в качестве аргументов команды. Здесь мы создали пользователя admin с паролем admin.
В FS мы вошли в качестве этого пользователя с помощью команды enter.
Мы оказались в корневой директории. Это же может подтвердить команда pwd.
Команда ls показывает содержимое текущего каталога: пока оно пустое.
Создадим папку с помощью команды mkdir.
Теперь ls покажет, что эта папка создана.
Аналогично создадим y, ls это отобразит.
Перейдём с помощью команды cd в x.
С помощью команды load загрузим с файловой системы ПК файл temp.txt, поместим его в текущую директорию (/x/) с именем temp. ls покажет, что это действительно произошло. Команда cat выведет содержимое файла. Оно совпадает с исходным.

2)
Скопируем temp с помощью команды copy в a. С помощью ls и cat покажем, что копирование произошло успешно.
Переместим temp в b. Перемещение произошло успешно.
Скопируем b в /y/x

3)
Проверим, что в y есть файл x.
Команда cdback возвращается в родительский каталог.
Сохраним x на ПК в файл x.txt. Это действие прошло успешно, в каталоге с запускаемой программой можно найти этот файл, имеющий идентичное содержимое.

4)
Скопируем /y/z в /x.
Создадим пользователя с логином usr и паролем usr.
Дадим ему доступ (getacc) к файлу z.
Дадим полный доступ (getfullacc) к папке /x (т.е. к ней и к файлам в ней).
Выйдем с помощью exit.
Сохраним в FS состояние файловой системы в файл save1 (store).
Войдём в usr usr.
Увидим, что в его версии файловой системы есть доступ к /x и двум файлам в ней, и к файлу z.
Удалим доступ пользователя admin к файлу /x/a.
Удалим файл z. Убедимся, что он удалён.
Выйдем.

5)
Вновь войдём как admin. Увидим, что есть доступ к x c одним(!) файлом и y. Зайдём в /x/, там действительно есть доступ только к b.
Выйдем из пользователя и из текущей FS.
Загрузим состояние FS из save1.
Снова зайдём в admin.
ls покажет, что состояние осталось таким, каким оно было до выгрузки fs.
